## 決策

這場是演唱會票券，流程採用本人帳號、本機瀏覽器、官方頁面與平台規則內的準備流程。Tickets Hunter 的研究用途是把正確活動頁、會員登入狀態、開賣等待 gate、設定檔與本機 Chrome 啟動路徑先確認好；驗證、付款與最終訂單確認仍以本人當下操作與官方頁面為準。

## 明天操作流程

### Tickets Hunter 設定

- `homepage` 放一般票場次頁：
  `https://ticket.com.tw/application/UTK02/UTK0201_00.aspx?PRODUCT_ID=P1BT1FIT`
- 不要把 `homepage` 設成年代首頁或會員登入頁；程式會在有年代帳號時先導到登入頁，登入後再回到設定的活動頁。
- `refresh_datetime` 放開賣時間：
  `2026/06/24 12:00:00`
- 只啟動一個搶票流程；重複按「搶票」會產生多個 `nodriver_tixcraft.py` 子程序，反而讓瀏覽器狀態難以判斷。
- Chrome for Testing 下載到 `src/webdriver/` 是本機 runtime 產物，不提交到 repo。

### 11:00-11:30

- 開啟 Tickets Hunter 設定頁，確認年代帳號與密碼已存檔。
- 按一次「搶票」，確認 Chrome 開到年代登入或活動頁。
- 登入年代會員，確認 Email/手機/會員驗證已完成。
- 確認信用卡可刷海外或網路交易，3D 驗證手機在手邊。
- 先決定可接受票價排序，例如：`6280 -> 5980 -> 4980 -> 4280 -> 3880 -> 3280`。
- 若要買身障、輪椅或陪同票，先確認資格；不符合資格就只買一般票。

### 11:55

- 保留一個主要購票視窗即可，不要多開或高頻手動重整。
- 停用會擋彈窗或干擾付款的瀏覽器外掛。
- 把信用卡、手機 OTP、收件資料準備好。

### 12:00

- 手動重新整理官方一般票頁。
- 點「立即訂購」。
- 手動選場次、區域、票數與座位。
- 手動輸入驗證碼或安全驗證。
- 手動確認訂單內容與付款；看到訂單編號才算完成。

## 使用邊界

- 不保存年代帳號密碼、Cookie、信用卡或身分資料到 repo。
- 不把訂單、付款、信用卡驗證或平台回傳的私密畫面提交到 repo。
- 不轉售、不代購加價、不用虛偽資料。
- 若明天官方頁臨時更新規則，以官方頁面當下內容為準。
